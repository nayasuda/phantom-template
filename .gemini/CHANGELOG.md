# Phantom 変更履歴

## 2026-02-15: ブランチフロー最適化（重要）

### 何が変わった？
全エージェントの設定ファイルを更新し、**ブランチ作成・PRフローを「反映先」で判断する**ように変更した。

### 3つの作業モード（モナが判断してメンバーに指示する）

| 反映先 | 作業モード | やること |
|--------|-----------|---------|
| main のコード変更（scripts/, .gemini/, .github/ 等） | PRフロー | ブランチ→実装→PR→レビュー |
| 外部デプロイ（GAS/clasp push 等） | 直接コミット | main で直接コミット→デプロイ |
| docs下書き（NOTE_DRAFT_*.md 等） | 直接コミット | main で直接コミット |
| コード変更なし（Project管理、調査） | Git操作なし | ブランチもコミットも不要 |

### draft item 運用ルール（明確化）
- draft item = ジョーカーとエージェントの共有タスクボード
- コード変更を伴わないタスクはすべて draft item で管理
- Issue は「main コード変更+PR必要」「外部共有必要」「ジョーカー明示指示」の3パターンのみ
- **同じタスクで draft item と Issue の両方を作るな（二重登録禁止）**
